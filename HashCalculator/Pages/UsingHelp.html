主要功能介绍
此程序主要是为了批量计算、批量校验文件的哈希值而设计，支持计算文件的 SHA1、SHA256、SHA384、SHA512、MD5 等哈希值。
适用用户：需要快速对一批文件生成哈希值并发布、需要快速校验一批文件的哈希值是否与文件发布者公布的哈希值一致。此程序当然也支持单个文件的哈希值的计算、校验，但是不会显示单个文件的大小、计算耗时等详细信息。

常用功能使用方法
  • 批量计算文件哈希值
    1. 可选步骤：在右上角的"算法"下拉选择框中选择您要计算的哈希值类型。
    2. 必选步骤：选定要计算哈希值的文件。将多个文件或文件夹拖入主界面的表格中，或者点击右上角"选择文件"按钮选择文件。
    3. 等待结果：上一个步骤选择的文件会被自动添加到主界面的表格中等待被计算，计算完成后会在表格的"哈希值"列中显示计算的哈希值类型和结果。
    4. 可选步骤：点击"导出结果"按钮。完成全部文件的计算后，此按钮变为可用状态，点击后会弹出储存位置选择对话框，选择合适位置保存即可，表格中"导出"列被勾选的行的哈希值和文件名都将被导出到文本文件中。
    5. 可选步骤：点击"校验"按钮。如果你在第 2 个步骤中选择的是别人发布的文件，而你想要校验你下载的这些文件与别人公布的文件哈希值是否一致，那你可以：
      • 对于单个文件：复制网页上公布的哈希值，粘贴到本程序的"校验依据"输入框中，然后点击后面的"校验"按钮，表格中的"校验结果列"将会显示代表校验结果的颜色。
      • 对于多个文件：文件的发布者一般会随着文件发布一个包含文件哈希值和文件名的文本文件，点击"校验依据"后的"..."按钮，选择该哈希值文件，或将该哈希值文件直接拖入"校验依据"输入框，然后点击后面的"校验"按钮，表格中的"校验结果列"将会显示代表校验结果的颜色。更快速的多个文件校验方法：快速批量校验文件哈希值。
      • 请注意！！！ 该包含"文件哈希值-文件名"的文本文件内容必须是通用排版格式，即每行都是 6b86b273ff34fce19d6b804eff5a3f5747ada4eaa22f1d49c01e52ddb7875b4b *文件名 这样的形式，否则本程序将无法解析该哈希值文件的内容并完成校验。其中哈希值不区分大小写，星号可有可无。
    6. "校验结果"列显示的颜色的含义
      • 校验结果显示色：该文件的哈希值与"校验依据"中的哈希值一致。
      • 校验结果显示色：该文件的哈希值与"校验依据"中的哈希值不一致。
      • 校验结果显示色：该文件与"校验依据"没有关联。
      • 校验结果显示色：程序未能确认被校验文件的哈希值是否与校验依据中对应文件名指示的哈希值是否匹配。出现此结果的原因是校验依据文件中含有相同的文件名，且这些相同文件名指示的哈希值不一样。
      • 校验结果没有显示颜色：该行文件的哈希值结果还没有执行过校验操作。
    7. 查看校验报告：完成文件的哈希值计算后或者点击"校验"按钮后，将鼠标悬停在任意一行的"校验结果"列上，就会显示本次校验结果的汇总报告。
  • 快速批量校验文件哈希值
    当你需要批量校验已下载的一批文件的哈希值时，如果这些文件的发布者提供了包含这些文件哈希值的文本文件(下称 sums 文件)，且该哈希值文件是通用格式（参考以上请注意！！！处的描述）的文本文件，则可以使用此功能。使用此功能需要保证面板为空，如果表格有内容，可以通过"清空面板"按钮清空表格。
    1. 可选步骤：在右上角的"算法"下拉选择框中选择您要计算的哈希值类型。
    2. 必选步骤：将 sums 拖入"校验依据"输入框中，或者点击输入框后的"..."按钮选择 sums 文件，然后点击"校验"按钮，程序会自动在 sums 文件同一目录下查找相应要被校验哈希值的文件，然后显示在表格中并完成计算、显示校验结果。
    3. 查看校验报告：参考查看校验报告处的查看方法描述。

各按钮及设置项的功能描述
  • 主界面各按钮作用
    1. 菜单栏的"设置"菜单：打开程序设置界面。
    2. 菜单栏的"帮助"菜单：打开此帮助界面。
    3. 菜单栏的"窗口置顶"复选框：勾选则使程序窗口保持在其他窗口的上方，即使已经切换到其他程序，本程序窗口也不会被其他窗口遮挡。
    4. 菜单栏的"算法"下拉选择框：要计算文件的哈希值类型。
    5. 菜单栏的"选择文件夹"按钮：选择要被计算哈希值的一批文件所在的文件夹。
    6. 菜单栏的"选择文件"按钮：选择要被计算哈希值的文件。
    7. 主界面下方的"校验依据"输入框：用于输入要校验的哈希值或者 hashsums 文件的路径。
    8. 主界面下方的"..."按钮：选择 hashsums 文件并将路径显示在"校验依据"输入框。
    9. 主界面下方的"校验"按钮：以"校验依据"输入框的哈希值或 sums 文件内的哈希值为准，与表格中的文件计算出来的哈希值对比并将结果显示在表格的"校验结果"列。
    10. 主界面左下方的"完成进度"进度条：正在计算文件哈希值时才会显示，表示已选择的所有文件的哈希值计算完成进度，此进度是以文件数为单位。
    11. 主界面下方的"清空面板"按钮：将表格中的所有计算结果清空。
    12. 主界面下方的"导出结果"按钮：将表格中所有"导出"列已勾选的行的哈希值和文件名导出为一个通用格式的文本文件。
    13. 主界面下方的"取消任务"按钮：立即取消所有已添加到表格中的哈希值计算任务并停止向表格增加剩下的任务，已完成的任务不受影响。
    14. 主界面下方的"新行重算"按钮：以"算法"下拉选择框选择的算法为准，计算表格中所有文件的哈希值，但不清除原来的计算结果，而是增加新的行来显示结果。这对于需要对同一个文件计算多个类型的哈希值时很有用。
    15. 主界面下方的"原地重算"按钮：以"算法"下拉选择框选择的算法为准，清空表格内容并重新计算表格中所有文件的哈希值。
  • 设置面板各项作用
    1. 主窗口关闭时记住窗口位置：下次打开程序时从上次关闭时的位置显示窗口。
    2. 主窗口关闭时记住窗口大小：下次打开程序时以上次关闭时的大小显示窗口。
    3. 当拖入面板的对象包含文件夹时：拖入面板的对象包含文件也包含文件夹时，正常计算文件的哈希值，文件夹则使用以下规则：
      • 搜索一代子文件：对于拖入面板的文件夹，只搜索其直接子文件，而不搜索该文件夹内的文件夹。
      • 搜索全部子文件：对于拖入面板的文件夹，深入搜索其内的所有层级文件夹，获取其内的所有文件。
      • 不搜索该文件夹：如果拖入面板的对象是个文件夹，则略过该拖入的文件夹。
    4. 计算所得哈希值使用小写字母形式：主界面表格中的"哈希值"列显示的哈希值字母使用小写形式，否则使用大写形式（大小写不影响校验功能）。
    5. 同时计算多少个文件的哈希值：选择合适的选项可以加快文件哈希值的计算速度，这里的"大小"是相对于硬盘的读取速度来说的。
      • 1 个 - 大多数文件很大：当大多数拖入面板或选择的文件都很大（500MB 以上）时使用此选项。
      • 2 个 - 大多数文件较大：当大多数拖入面板或选择的文件都较大（300MB 左右）时使用此选项。
      • 4 个 - 大多数文件较小：当大多数拖入面板或选择的文件都较小（100MB 左右）时使用此选项。
      • 8 个 - 大多数文件很小：当大多数拖入面板或选择的文件都很小（30MB 及以下）时使用此选项。
    6. 使用快速校验时：
      • 搜索依据所在目录的一代子文件：只搜索与 sums 文件同级文件夹的文件。
      • 搜索依据所在目录的全部子文件：搜索 sums 文件所在文件夹下所有层级的文件。
    7. "默认"按钮：读取默认设置并将设置面板各项设置重置为默认选项，但设置并不会生效。
    8. "关闭"按钮：不应用已修改的设置，直接关闭设置面板。
    9. "应用"按钮：使设置面板的设置生效，设置面板关闭后会将已生效的设置保存为本地文件。

可能出现的异常的含义
  • 复制哈希值时弹出窗口显示"复制哈希值失败"：点击复制按钮过快或者偶发复制失败。
  • 弹窗显示"意外的异常，可打开帮助页面末尾链接向开发者反馈"：程序意料之外的异常，可以到 Gitee Issues 或 GitHub Issues 或 吾爱破解论坛 向开发者反馈问题（需要登录）。
  • 在依据所在文件夹中找不到依据中列出的文件：请确认依据文件 sums 是否与要被计算哈希值的文件在同一个文件夹。如果要被计算哈希值的文件在比 sums 更深层的文件夹中，则请打开"设置"，检查"当使用快速校验时"是否选择了"搜索依据所在目录的所有子文件"选项。
  • 要计算哈希值的文件不存在或无法访问：可能因为用户权限原因无法访问该要被计算哈希值的文件。
  • 读取要被计算哈希值的文件失败或计算出错：打开要被计算哈希值的文件失败，或者计算该文件哈希值的过程中出现不可预料的错误。
